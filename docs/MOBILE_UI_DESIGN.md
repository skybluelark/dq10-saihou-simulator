## 4. 素材方針(生成画像)

### 4.1 パーツ分割の設計観点(差し替え考慮)

素材は「画面一枚絵」ではなく**部品単位**で管理し、個別差し替え・状態差分・多解像度化に耐える構成とする。

- **背景**: 画面全体の背景(セーフエリア外まで敷く)。画面種別ごと(プレイ画面/結果画面など)
- **パネル枠**: 状態表示・ログ等の枠。**9-slice(周囲を固定し中央を伸縮)**で可変サイズに対応できる形が望ましい
- **ボタン**: 特技ボタン・汎用ボタン。通常/押下/無効/選択中の**状態差分**をセットで用意。文字は画像に焼き込まず**テキストを重ねる**(特技追加・文言変更・多言語化に耐える)
- **マス**: 布マスの下地(状態色: 白/黄/緑/赤 は下地差分 or 色調変化のどちらで表すか)。しつけ枠・発光・対象ハイライトは**重ねる装飾パーツ**として分離
- **アイコン・バッジ**: ぬいパワー、布特性、必殺チャージ、無我など
- **エフェクト**: 吹き出し(ダメージ/会心/回復)、必殺演出など(静止画+CSS/コードアニメーションか、連番/スプライトか)

命名・格納規約(assets/ 配下のディレクトリ構成、状態差分のサフィックス等)は画面設計の確定後に定める。

### 4.2 権利面の線引き

- DQ10 の**画像・ロゴ・フォント・スクリーンショットの流用は不可**(著作権)。生成画像でも「DQ10 の UI を忠実に再現したもの」は依拠性が認められ得るためリスクがある。
- 「木枠+布地風」「和風RPGメニュー調」のような**雰囲気のレベルで寄せる**のは一般に問題ない。固有のデザイン(DQ特有のウィンドウ枠・書体・配色の組合せの再現)は避ける。

### 4.3 DQ10寄せの見え方確認(P3)— 決定済み

モック第1版(テーマA=モダン/テーマB=DQ雰囲気寄せ)の比較により、**トーンB(DQ雰囲気寄せ: 紺ウィンドウ+金縁+羊皮紙調。固有デザインの再現はしない)で決定**(2026-07-08)。以降の素材生成プロンプトはトーンBを基準にする。

### 4.5 命名・格納規約(2026-07-08 起票)

- 格納: `assets/ui/<カテゴリ>/` (カテゴリ = bg / panel / btn / cell / icon / badge / fx)
- 命名: `<ID>_<状態>@2x.png`(状態: normal / pressed / selected / disabled / on / off。単一状態は状態部を省略)
- 透過PNG。9-slice の切断位置はパーツごとに一定(角24px@2x を標準)とし、実装形式(W1 の A8 確定後)に合わせてメタデータ化する

### 4.6 生成プロンプト一覧(トーンB基準。§4.4 の全パーツを画像AIで作成: 2026-07-08 決定)

#### 4.6.1 生成の運用ルール

- **共通スタイル(全プロンプトに前置)**: 「和風ファンタジーRPGのゲームUI素材。深い紺色の布地に白の縁取りと金の飾り罫。羊皮紙と織り布・ステッチ(縫い目)の質感。フラットで視認性が高い。単品・正面。**背景はパーツに使われていない単色(例: 純緑 #00FF00)でベタ塗り**(透過や市松模様は指定しない — 市松は描き込まれてしまうため。単色背景はこちらで透過化する)」
- **個別プロンプトで色・縁・装飾を指定した箇所は、共通スタイルの配色(紺・金)より個別指定を優先する**(生成テストで共通スタイルの紺・金装飾が個別指定を上書きする傾向が確認されたため。2026-07-09)
- **文字・ロゴは一切入れない**(日本語文字の生成品質が低かったため例外なしで確定。テキストはすべて実装側でフォント合成する。2026-07-08)
- **px は最終使用サイズ**。同じ縦横比で **2〜4倍の大きさで生成してよい**(縮小・切り出し・透過化はこちらで実施)
- 9-slice 対象(枠・ボタン類)は「**コーナーの装飾は端から辺長の1/6以内に収め、四辺の中間は伸縮しても破綻しない単純な繰り返しにする**」を個別プロンプトに含めている
- 1画像1パーツを推奨(シートにまとめる場合は各パーツの間隔を広めに)

§4.4 の全パーツ(fx_hissatsu を除く42点)をユーザー生成の画像AI素材に置き換え、モック v2.4 に組込み済み。

- **出力**: `public/mock/assets/<ID>.png`(§4.4 の最終px。**9-slice 対象は生成物の縦横比を保持して高さ基準で出力**(幅可変)— border-image で伸縮するため装飾が歪まない)
- **スライス値(@2x)**: panel_window=96(表示: 小型パネル10px/結果窓28px)、panel_grid=96(表示16px。2026-07-12: 出力512化に伴い変更。旧24/表示12px)、btn_skill/finish=26(表示8px)、btn_primary=50・btn_secondary=45(表示 縦20px・横14px)。マス・バッジ類は背景表示(100%伸縮)。**chip は横方向のみ3分割**(border-image slice `0 32 fill`、キャップ表示幅=表示高の1/2): 背景100%伸縮だと幅の異なるチップで両端の曲率が変わるため、左右キャップの縦横比を固定し中央だけ横伸縮する(2026-07-10 実機レビュー反映)
- **透過化の方式**: 境界・グロー帯は緑過剰(G−max(R,B))によるα推定+アンミックス。布地内部は3px侵食マスクでα=1保護(緑系布地の変色防止)。広い発光を持つ star_result_on/off・badge_sparkle は全域クロマα
- **緑相ワイプ**(2026-07-10 追加): 正当な緑を含まない12点(cell系5種・btn_skill系4種・btn_finish系2種・icon_power)は、緑背景のビネット・焼き込み影の明度ムラが「背景色からの距離」判定をすり抜けて半透明ヘイズ/緑残りになるため、**色味比率 greenness=(G−max(R,B))/max(R,G,B)** で判定し、greenness>0.35 は侵食保護より優先して α=0 に確定、中間帯(0.12〜0.35)はアンチエイリアスとして半透明化する。角丸の外側は完全透過になる(実機レビュー指摘の対応)
- **白リム剥離**(2026-07-10 追加): btn_skill系4種・btn_finish系2種は生成時に**全周へ純白アウトライン(最外周・四隅では弧沿いに幅広)が焼き込まれて**おり、角の外側は透過済みでも弧上の白がα≈1で残るため「角が透過されていない」ように見える(実機レビュー指摘)。透明領域に接する「ほぼ無彩色の白」(unmix後RGBで min≥180 かつ max−min≤50)を外周から連鎖的に剥離してα=0にする。内側の装飾(金=黄寄り・紺=青寄り)は彩度で連鎖が止まる。btn_skill_pressed のみリムが暗いグレーに落ちているため中明度の完全無彩色グレー(min≥90 かつ max−min≤20)も対象(※disabled はグレーアウト本体が無彩色のため白判定のみ)。btn_header は最外周が暗色のため対象外。**仕上げとしてリサイズ後の最終出力解像度でもハロー除去を行う**(2026-07-10 追加): ソース段階の剥離は列ごとに停止位置が揺れ、LANCZOS縮小が取り残しを外周へ半透明の白/明灰として滲ませる。@2x表示では9-sliceコーナーがほぼ等倍で描画されるため紺縁の外の明るい画素は1個でも視認される(「btn_undo と同水準なら許容」の当初判定は誤り — btn_undo は明るい金縁の外の白で目立たないが、こちらは**暗い紺縁の外の白**でコントラストが立つ)。透明領域(画像外含む)に接する明るい無彩色画素(min≥150 かつ max−min≤80)を最終画素で連鎖除去。選択(金)ボタンのみ縁自体が明るいため純白寄り(min≥200 かつ max−min≤45)**+明るい完全無彩色(min≥150 かつ max−min≤25。下辺に残る影混じりの白リム=実測(199,198,188)α1.0 の1pxライン対策)**に限定し、クリーム〜淡金の暖色の縁(max−min≥50)は残す
- **影の付け方**(2026-07-10 追加): 角丸で角が透明な素材(ボタン類)には **box-shadow を使わず `filter: drop-shadow()` を使う**。box-shadow は要素の矩形に沿って描かれるため、角丸の外側の透明領域が「縁の弧/明るい下地のくさび/矩形影の角」に挟まれて白残りのように浮く(実機レビューの btn_skill 系「角の白残り」の最終的な真因。btn_undo が問題なく見えたのは影指定がなかったため)。drop-shadow は画像のアルファ形状に沿う。パネル類(panel_window/panel_grid)は角がほぼ不透明(実測 α平均0.89)のため box-shadow のままで実害なし
- **布グリッド枠の外周緑ヘイズ除去**(2026-07-12 追加): panel_grid_normal/light/rainbow は出力画像の**画像端(外周1〜2px)に純緑背景のヘイズ**(半透明の緑被り α0.4〜0.6・greenness 0.3〜0.66)が残り、bg_main との境界で緑フリンジとして視認される(実機レビュー指摘)。出力解像度で**画像端に連結した緑味画素(greenness>0.25)を連鎖除去**する(GREEN_EDGE_PARTS)。~~panel_grid_regen は対象外~~ → **regen も対象に変更(2026-07-12 実機レビュー: モックは再生布=regen を表示しており緑残りが視認された)**。方式は**帯域方式に確定(2026-07-12 改定)**: コンタミはシルエット境界の現象のため、**「透明領域(画像外含む)から6px以内の帯域」×「色条件」**で除去する(当初の「画像端/透明に連結した緑の連鎖除去」は、512px出力で緑帯が透明リングから浮いて数千pxを取り逃したため置換え)。色条件は通常3種=greenness>0.25、**regen(緑枠)=明るい純緑(G≥140 かつ greenness>0.40)のみ**(枠本体の暗い緑・縁のAAは残す)。実測根拠: 4種とも該当色の画素は帯域外に0で誤除去なし。処理後実測: 4種とも帯域内外の該当緑0、regen の正当な緑は無傷。**透過色を変えた再生成は不要**。

- **正円ボタンのトリム**(2026-07-11 追加): btn_step_* は円の上下端・左右端で行/列の被覆率が薄く、標準しきい値(15%)のトリムでは縁が切れて非正方形クロップ→88×88圧縮で楕円化していた(実測: 真円550×551に対し15%トリムは538×511)。**低しきい値(2%)でトリムし、中心維持で正方形化+片側4%のマージン**を付けて切り出す(ROUND_PARTS)。処理後実測: 円81〜82px・縦横差1px以内・全周マージン3〜4px
- **左右反転による派生出力**(2026-07-10 追加): btn_step_redo_* は原本を持たず、**btn_step_undo_* の原本を水平反転してから同一パイプラインで処理**する(MIRROR_SRC)。生成AIに左右反転版を作らせると意図しない差異が入るため、プログラムで画素レベルの鏡像を保証する(スモークテストで undo出力の反転と redo出力の完全一致=差分0 を確認済み)
- **アルファブリード**(2026-07-10 追加): 上記の素材側対策後も実機で白フリンジが残った原因は、**透明画素(α≈0)のRGBが白系のゴミ値のまま残る**こと(事前乗算リサイズの逆算 premul/α が α≈0 でノイズを増幅する)。ブラウザが border-image を端末解像度へ再補間する際、straight-alpha 補間だと「透明白×不透明紺」の中間色=白フリンジが縁の外側に出る(デスクトップと実機でレンダラーの補間経路が異なるため実機でだけ目立つ)。対策として、出力の最終段で**α<10/255 の画素のRGBを最近傍の信頼できる色(α≥10/255)で反復的に埋める**。補間結果が縁色(紺/金)に収束するため白フリンジが出ない

### 5.12 シミュレーター設定画面(2026-07-11 確定)

- **入口**: レシピ選択画面のタイトル行右側(◀もどるの反対側)に、もどるボタンと**同型同サイズ**の「設定」ボタンを配置。押下で画面を切り替える。
- **画面タイトル**: 「設定」。
- 各設定は選択ボタン+**その下部に説明文言**の形で並べる:

  | 設定 | 選択肢(太字=デフォルト) | 説明文言 |
  |---|---|---|
  | Lv設定 | **なし**/あり | 職人レベルを変更するか設定します。変更しない場合はLv80前提となります。 |
  | コツ設定 | **なし**/あり | コツの取得有無を切り替えられます。設定しない場合はコツあり前提となります。 |
  | 数値の表現 | **残数値**/累積ダメージ | グリッドに表示する数値の意味合いを設定します。 |
  | 基準値の表示 | **なし**/あり | グリッドの数値の下に、基準値を表示するか設定します。 |
  | 特技ボタン配置 | (下記) | 位置を入れ替えたいボタンを続けてタップしてください |

- **特技ボタン配置(C2)**: 設定画面の**最下部**に、プレイ画面と同じ5行4列の位置関係で**無効色のボタン群**を配置。1つ目のタップで**選択色**に変更、別のボタンをタップすると**位置を入れ替え**、選択済みボタンを再タップすると**未選択に戻る**。**「しあげる」も入れ替え対象に含む**(20個すべて対象。2026-07-11 確定。§7-12 解決)。
- **Lv設定/コツ設定「あり」時のレシピ選択画面**: さいほう針の設定部分の**上側に設定領域を1行追加**し、その分レシピ一覧が縦方向に短くなる。**同じ行に左=Lv設定、右=コツ設定**:
  - **職人レベル**(文言): 数字の左右にボタンを2つずつ(±1・±10)。デフォルト80、可変範囲**1〜80**。±10で範囲を超える場合は**最小/最大値へクランプ**、すでに端にある場合は**ボタンを無効化**。
  - **コツ**(文言): できのよさと同様の「あり」「なし」選択。デフォルト**「あり」**。
  - 開始集中力の表示はLvに連動して再計算する(SPEC §3.5 レベル別基礎値)。
- **Lv設定のゲーム動作への影響**(SPEC v1.24/v1.25 §1.1 改定): 開始時集中力の基礎値テーブル参照に加え、**習得Lv未満の特技を使用不可(無効状態表示)**とする。会心率パッシブはLv10のみ実効(SPEC §6-1 の仮定)。**Lv40未満では必殺チャージは一切発生しない**(Lv40のイベント習得のため。開幕チャージ含む。SPEC §3.3)。必殺率小アップ(Lv45/57/62)は効果不明のため反映しない(SPEC §6-10。§7-15 解決)。

### 5.13 ログ画面(2026-07-11 確定)

- **入口**: プレイ画面ヘッダー右上ボタンの3状態ループ(§5.10)。
- **切替範囲**: **特技ボタンの表示部分**をログ表示領域に入れ替える(ステータス領域は残り、**1手戻す/進むは有効**)。
- **ログ内容**: 行=**ターン・特技・ぬいパワー・位置・ダメージ**の5項目(2026-07-12 実機レビューで確定)。**出目(実ロール値)は表示しない。会心率は管理者モードのみ**(D31 改定)。領域に収まらない分は**縦スクロール**。
  - **位置**: グリッドの慣例表記=**左上から横方向、端まで行ったら次の行、の順のアルファベット**(3×3=A〜I、2×3/3×2=A〜F、2×2=A〜D)。複数マスヒットは**ダメージの値と順序を対応させて「/」区切り**で並べる(例: 位置「E/B/D/F/H」・ダメージ「9/6/6/7/6」)。
  - **ぬいパワー**(特技名と位置の間の列): そのターンに実行されたぬいパワーを表記。**「？」から変化したターンは「？→弱い」の形式**。**精神統一による固定中のターンは固定されたぬいパワーをそのまま表記**する(「？」から変化したターンで精神統一した場合も、固定中の後続ターンは固定パワーのみ。**？→X 表記が付くのは精神統一を使ったそのターンだけ**)。
  - **ダメージ欄は左寄せ**(「/」区切りで位置と対応させて読むため。2026-07-12)。**糸ほぐしのダメージは負数表記**(数値表・ログ共通。2026-07-12)。列間の余白は特技名列が過大にならないようバランスを取り、**特技列以降の列間隔は1文字分程度ずつ広めに**とる(2026-07-12)。**会心は「(会心)」等の文言を付けず、文字色(オレンジ)のみで表現**する(複数マスヒットで文言があるとはみ出すため。会心はマスごと独立のため、**複数ヒット時は該当する値のみ着色**。2026-07-12)。
- **現在の履歴位置のターンをハイライト**(D30。ブラウザ版と同様)。1手戻す/進むの押下でハイライト位置も連動する。
- **ログ表示中は布グリッドのタップと、ぬいパワー行の表示切替(§5.1.1 のその場トグル=サイクル全体表示への切替)を無効化**する。

### 5.14 リプレイコード入力画面(2026-07-11 確定)

- **テキスト入力領域**(リプレイコード)+その下に**「クリップボードから貼り付け」ボタン**(コピペ入力を想定)。
