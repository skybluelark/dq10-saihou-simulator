## 4. 素材方針(生成画像)

### 4.1 パーツ分割の設計観点(差し替え考慮)

素材は「画面一枚絵」ではなく**部品単位**で管理し、個別差し替え・状態差分・多解像度化に耐える構成とする。

- **背景**: 画面全体の背景(セーフエリア外まで敷く)。画面種別ごと(プレイ画面/結果画面など)
- **パネル枠**: 状態表示・ログ等の枠。**9-slice(周囲を固定し中央を伸縮)**で可変サイズに対応できる形が望ましい
- **ボタン**: 特技ボタン・汎用ボタン。通常/押下/無効/選択中の**状態差分**をセットで用意。文字は画像に焼き込まず**テキストを重ねる**(特技追加・文言変更・多言語化に耐える)
- **マス**: 布マスの下地(状態色: 白/黄/緑/赤 は下地差分 or 色調変化のどちらで表すか)。しつけ枠・発光・対象ハイライトは**重ねる装飾パーツ**として分離
- **アイコン・バッジ**: ぬいパワー、布特性、必殺チャージ、無我など
- **エフェクト**: 吹き出し(ダメージ/会心/回復)、必殺演出など(静止画+CSS/コードアニメーションか、連番/スプライトか)

命名・格納規約(assets/ 配下のディレクトリ構成、状態差分のサフィックス等)は画面設計の確定後に定める。

### 4.2 権利面の線引き

- DQ10 の**画像・ロゴ・フォント・スクリーンショットの流用は不可**(著作権)。生成画像でも「DQ10 の UI を忠実に再現したもの」は依拠性が認められ得るためリスクがある。
- 「木枠+布地風」「和風RPGメニュー調」のような**雰囲気のレベルで寄せる**のは一般に問題ない。固有のデザイン(DQ特有のウィンドウ枠・書体・配色の組合せの再現)は避ける。

### 4.3 DQ10寄せの見え方確認(P3)— 決定済み

モック第1版(テーマA=モダン/テーマB=DQ雰囲気寄せ)の比較により、**トーンB(DQ雰囲気寄せ: 紺ウィンドウ+金縁+羊皮紙調。固有デザインの再現はしない)で決定**(2026-07-08)。以降の素材生成プロンプトはトーンBを基準にする。

#### 5.1.1 サイクル全体表示の切り替え方式(決定: 2026-07-08)

**案A「パワー表示行のその場トグル」を採用** — D2/D3 のパワー表示行全体をタップ可能にし、タップで「現在+次3つのアイコン」⇔「サイクル全体(現在位置ハイライト+布特性予告つき)」を同じ領域で切り替える。行の右端に小さな開閉アイコン(シェブロン等)を置いてタップできることを示す。別レイヤーを持たず高さも固定できるため部品が最少。

- **サイクル全体表示は最大10個になるため、左右スライドで横スクロールできるようにする**(2026-07-08)。
- パワーアイコンへの実行ターン番号(T番号)の併記は**行わない**(2026-07-08)。
- 行の高さは控えめでよい(第1版モックより縮小)。
- 表示が小さくなりすぎる等の問題が生じた場合は再検討する(代替: 案B=行直下へのポップオーバー表示)。

#### 5.1.2 サイクル表示での布特性予告の表現(検討中)

ブラウザ版の**下線**(実線/点線・色分け。SPEC §4.3)はブラウザ版の実装優先で決めた仕様であり、**スマホ版では変更する**(2026-07-08 決定)。D9 廃止により布特性の事前情報はこの表示に集約されるため、下線より読み取りやすい表現とする。候補:

- **案1(推奨): 特性アイコンバッジ** — サイクル表示時、発動ターンに当たるエントリへ**布特性アイコン**(再生=緑 / 虹・半減=水色 / 虹・会心=黄 / 光=ピンク)を重ねる(角バッジ等)。次の周回で到達する分(ブラウザ版の点線相当)は**半透明バッジ**で表す。※当初案にあった実行ターン番号の併記は不要と判断(2026-07-08)。
- **案2: アイコンの色枠/グロー** — 発動エントリのパワーアイコンを特性色の枠や発光で囲む。省スペースだが D5(精神統一の淡青発光)と表現が近く、混同に注意。
- 最終決定はモック段階で見た目を比較して行う。

#### 5.4.1 リプレイモード(方針決定: 2026-07-08、方式は検討中)

**スマホ版では検証モードの選択(トグル)は設けない**。代わりに**リプレイモード**を用意する:

- リプレイモード中は、**特技欄の領域とトグルで行動ログを表示**する方式を検討する(通常プレイのログ呼び出し(D29)とは別に、リプレイ追跡用の表示)。
- 現在ターンのハイライト(D30)はこのログ表示で使用する。
- **入り口**: シミュレーターの起動(レシピ選択画面)とは別に、**リプレイモードの入り口**を用意し、リプレイ読込はそこで行う(D38)。
- **操作**: 1手進むはリプレイモード限定。1手戻すは通常プレイでも常設(D37)。
- シードはスマホ版では表示・指定とも設けない(D36。再現はリプレイデータ経由)。
- 結果画面の誤差内訳表(D41)の扱いは §5.6 の判定で確定する。
