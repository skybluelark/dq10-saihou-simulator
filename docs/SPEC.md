## 1. 目的とスコープ

- DQ10 のさいほう職人ミニゲームをブラウザ上で再現し、以下を可能にする:
  1. **練習モード**: 実ゲームと同じ挙動での手動プレイ練習(素材を消費せず縫い方を試せる)
  2. **検証モード**: 乱数シードや会心率などを固定・表示し、手順の良し悪しを比較できる
  3. (将来)**最適化モード**: 指定レシピに対する期待値最大の手順探索
- 対象バージョン: 現行バージョンの仕様を基準とする(バージョン差分はデータファイルの差し替えで吸収できる構造にする)。
- 非スコープ(初期): 職人レベル上げ効率、金策計算、他職人(鍛冶・木工等)。

### 1.1 実装前提

シミュレータ実装にあたっての固定の前提。ゲーム仕様の不明点に対する仮定(§6)とは区別する。

1. **職人レベルは80固定**(開始時集中力の基礎値=207)。レベル入力UIは設けない。
2. **職人レベルによる特技の使用制限は設けない**(全特技を使用可能)。特技の習得Lvは将来のLv制限機能に備えデータとして保持する。
3. **「さいほうのコツ」は取得済み**(会心率+1%固定)。
4. **パッシブ(会心率アップ Lv10/20/30、必殺率アップ Lv45)は取得済み**。
5. **針(種類・★数)はユーザーが設定で選択する**(設定項目はこれのみ)。

| 用語 | 意味 |
|---|---|
| ゲーム | 1回の製作。開始から「しあげる」または失敗まで(旧称「セッション」。コード上の識別子は session のまま) |

| ランダム会心 | ぬいパワー「？」からランダムに「会心×2」が出たターン |
| 必殺チャージ | 必殺技を使用可能な状態。開幕効果またはターン終了時の判定で付与 |

## 3. ゲーム動作仕様(モデル)

### 3.1 布とマス

- 装備ごとにマスのレイアウトと各マスの基準値が決まっている。マス数は 9(体上)/7(ぬいぐるみ)/6(体下・腕・ラグ)/4(頭・足) の4パターンで全て。
- グリッドの向きは部位ごとに固定: 足=2×2、**頭=2行×3列のうち4マス((1,2),(2,1),(2,2),(2,3)の凸形)**、体上=3×3、体下=3行×2列、腕・ラグ=2行×3列、ぬいぐるみ=3×3のうち7マス(欠け位置はレシピによる。例: ウェディの人形・男は r1c1/r1c3 が欠け)。
- **空きマス(頭の欠け位置・ぬいぐるみの欠け位置)もアンカーとして選択できる**。複数マス特技で対象範囲に存在するマスが1つ以上あれば、存在するマスにのみダメージを与えてターンが進む。単マス特技、または対象範囲に存在するマスが1つもない場合、そのタップは無効(行動不成立。ターン・集中力を消費しない)。
- 各マスの初期「残り数値」はレシピ固有。レシピごとのデータは外部ファイル(recipes.csv)に持つ。
- マスの残り数値はマイナス(縫いすぎ)になり得る。マイナス時は「糸ほぐし」で戻す。

### 3.2 ダメージ計算

**糸ほぐし以外:**

```
ダメージ = ((基礎値 × 特技倍率) × ぬいパワー係数) × マス補正
```

- 基礎値: 12〜18 の7通りからそれぞれ等確率(各1/7)で取得
- 特技倍率: 特技ごとの威力倍率(§3.3。例: 2倍ぬい=2、かげんぬい=0.5)

**糸ほぐし(負のダメージとして扱う):**

```
ダメージ = (基礎値 × ぬいパワー係数 × マス補正)   ※基礎値が負のため結果も負(=回復)
```

- 基礎値: -6〜-9 の4通りからそれぞれ等確率(各1/4)で取得
- 糸ほぐしは特技倍率を持たず、マス補正が括弧の内側に入る。丸めは共通ルールをそのまま適用する(`ceil(-x) = -floor(x)` により、回復量の絶対値は切り捨てに相当する)。
- **糸ほぐしに会心判定はない**。
- **回復上限**: レシピの初期状態(累積ダメージ0)で頭打ち。例: 初期状態から3だけ削ったマスにぬいパワー普通の糸ほぐしを行うと、出目(6〜9)にかかわらず3回復と表示される。

**ぬいパワー係数:**

| ぬいパワー | 係数 |
|---|---|
| 弱い | 0.5 |
| 普通 | 1 |
| 会心×2 | 1 |
| 強い | 1.5 |
| 最強 | 2 |
| ？ | そのターンの開始時(特技選択前)に他の5種いずれかへ等確率(各1/5)で変化(「会心×2」に変化した場合がランダム会心)。変化後に精神統一を使うと、その抽選後のパワーが3ターン維持される |

**共通ルール:**

- 各（）内の計算で小数点以下の端数が生じた場合、**正方向の最も近い整数に丸める**(正の無限大方向 = `Math.ceil`。例: 3.5→4、-3.5→-3)。括弧を閉じるたびに適用する。丸め規則は全特技共通。
- マス補正: 通常1。しつけがけ(§3.3)で2倍、光布の発光マス(§3.6)で2倍。両者は重複し、その場合4倍。

**みだれぬいの動作詳細:**

1. ランダムに異なる4か所のマスを選ぶ
2. 選んだ各マスに対し、2倍、1倍、1倍、0.5倍の特技倍率で乱数をもとにダメージ値を生成する(それぞれに会心判定あり)。マス補正(しつけがけ・発光)は**各マス自身のものが適用される**
3. **ダメージは生成したマスにそのまま適用される**。「大きい値から順に縫う」は会心の2倍適用後の値で降順にソートした**表示上の縫い順のみ**で、どのマスにどの値が入るかのロジックには影響しない

**複数マス特技の判定:**

- 複数マス同時縫い(ヨコ/滝のぼり/水平/たすき/巻きこみ/みだれ等)のダメージロール・会心判定はマスごとに独立。

**ライン系特技のアンカー自動置換 (2026-07-06 確定):**

実機では、ライン系特技でパターンがグリッド範囲からはみ出すアンカーは選択できず、範囲内に収まる位置へ置換される:

- 水平ぬい: 1列目・3列目を選ぶと**2列目**に置換
- 大滝のぼり: 1行目・3行目を選ぶと**2行目**に置換
- ヨコぬい: 3列目を選ぶと**2列目**に置換(2列グリッドでは2列目→1列目)
- 滝のぼり: 1行目を選ぶと**2行目**に置換
- たすきぬい: 1行目を選ぶと**2行目**に、3列目を選ぶと**2列目**に置換
- 逆たすきぬい: 1行目を選ぶと**2行目**に、1列目を選ぶと**2列目**に置換
- 巻きこみぬいは**すべての位置を選択可能**(布外へのはみ出しは無視)。みだれぬい・単マス特技も置換なし

一般則: 単マス・巻きこみぬい・みだれぬいを除く複数マス特技は、パターンがグリッド範囲に収まるようアンカーが自動置換される。

- 置換は**選択位置(アンカー)そのものに適用**する。UI上もアンカー表示(濃い青枠)は置換後の位置に表示され、以降の操作(再タップでの実行)も置換後のアンカーに対して行われる(例: 水平ぬいで1列目・3列目をタップしても、2列目をタップした場合と同一の表示・動作になる)。
- **置換が入るのは1タップ目(アンカー設定)のみ**。アンカー設定後、**実行(決定)となるのは青枠(濃い青のアンカー・薄い青の対象範囲)の位置をタップした場合に限る**。青枠外の位置をタップした場合は、置換して決定にはならず、新たな1タップ目(アンカーの取り直し)として扱う。
- 対象範囲の青枠は、パターン範囲内のグリッド位置すべてに表示する(**空きマスにも表示**。ダメージが入るのは存在するマスのみ)。

**必殺技「無我の境地」:**

- 必殺チャージ状態で使用可能。消費集中力は0。
- 使用すると1ターン消費し、それ以降の会心率に×2の補正がかかる(ぬいパワーは次のものに移動する)。
- ×2補正はゲーム終了まで持続する。
- 使用後はそのゲームが終わるまで再チャージなし(1ゲームで必殺は最大1回)。

**必殺チャージの発生モデル:**

- 判定タイミング: ダメージを与えたターンの終了時。そのターンの合計ダメージが大きいほど、また道具の会心率が高いほどチャージしやすい(正確な式・確率は未公表のため、以下は仮定モデル。§6-4)。

```
チャージ率(ターン終了時) = 必殺基礎値 × そのターンの与ダメージ合計 × (基礎会心率+道具のできのよさ)
```

- 必殺基礎値 = 0.00233(約1/430)。game-params.json で調整可能とし、実測データが集まり次第校正する。
- チャージ保持中は再判定なし。与ダメージ0のターン(精神統一・しつけがけ・糸ほぐし等)は判定なし。

### 3.4 会心

**会心発生時のダメージ:**

1. 乱数生成〜ダメージ計算(§3.2)までを通常どおり実施し、その値を2倍する
2. 2倍した結果、そのマスの累積ダメージが基準値を超える場合は、基準値と等しい累積ダメージになる値に変更する(=残り0で頭打ち)

- 会心は必ず残り0にするのではなく、2倍ダメージが残り数値以上のときに限り残り0で止まる。会心では縫いすぎ(マイナス)は発生しない。
- **残り数値が0以下のマス(縫い終わり・縫いすぎ)には会心判定自体が行われない**(実機確認 2026-07-07)。判定には行動開始時点の残り数値を用いる(みだれぬいを含む全ての複数マス特技は対象マスが相異なるため、同一行動内のヒットが互いの判定に影響することはない)。デバッグモードの会心率表示も付かない。

**会心率の計算式:**

```
会心率 = ((基礎会心率 + 道具のできのよさ + コツ + スキル) × ねらい倍率 + 固定値上昇)
         × 必殺補正 × シフト会心補正
```

| 項 | 値 |
|---|---|
| 基礎会心率+道具のできのよさ | 針テーブル(下表)の★別の値として合算で持つ |
| コツ | +1%(取得済み前提: §1.1) |
| スキル(会心率アップ Lv10/20/30) | 名目 計+0.6%だが数字通りには働かない。実効値+0.1%を仮定(§6-1) |
| ねらい倍率 | ねらいぬい使用時 7、通常時 1 |
| 固定値上昇 | 虹布の会心ターン +24%、光布の発光マス +24%(§3.6) |
| 必殺補正 | 無我の境地使用後 ×2、通常 ×1 |
| シフト会心補正 | シフト会心ターン ×2(必殺と併用で4倍)。ランダム会心は補正なし(§6-3) |

- 「会心×2」はシフト会心とランダム会心で**内部的に別のぬいパワー**(表示・パワー係数は同一、シフト会心補正の有無のみ異なる)。精神統一で固定した場合もこの区別が持続する(§3.3。実機確認 2026-07-07)。

|---|---|---|---|---|---|
| 銅のさいほう針 | +0 | 1.0% | 1.1% | 1.2% | 2.0% |
| 鉄のさいほう針 | +10 | 1.5% | 1.6% | 1.7% | 2.5% |
| 銀のさいほう針 | +15 | 2.0% | 2.1% | 2.2% | 3.0% |
| プラチナさいほう針 | +25 | 2.5% | 2.6% | 2.7% | 3.5% |

| 奇跡のさいほう針 | +50 | 3.3% | 3.4% | 3.5% | 4.3% |
| 光のさいほう針 | +45 | 3.6% | 3.7% | 3.8% | 4.6% |

- 会心率列は「基礎会心率+道具のできのよさ」の合算値。
- 道具の開幕特殊効果: 奇跡系 = 30%で開幕集中力+30、光系 = 10%で開幕必殺チャージ。

### 3.5 集中力

**開始時集中力 = レベル別基礎値 + 針の集中度(§3.4) + 開幕特殊効果(奇跡系+30、発動時)**

- 補正はこの3要素のみ(料理・装備等の補正はない)。
- 職人レベルは80固定(§1.1)のため、実際に使用する基礎値は207。テーブル全体は将来のLv制限機能に備えデータとして保持する。

| 11〜20 | 71, 74, 74, 77, 79, 82, 82, 85, 88, 88 |
| 21〜30 | 91, 94, 94, 97, 100, 100, 103, 105, 108, 108 |
| 31〜40 | 111, 112, 112, 114, 118, 121, 122, 122, 124, 128 |
| 41〜50 | 131, 133, 136, 138, 138, 141, 141, 143, 146, 148 |

- 集中力が消費コストに満たない特技は使用できない。全特技が使用不能になっても強制終了ではなく、「しあげる」のみ実行可能。しあげるは任意タイミングで実行できる(消費0)。

### 3.6 布の特性(素材ギミック)

レシピごとに布タイプが決まっている:

| 布タイプ | 挙動 |
|---|---|
| 通常(黄色) | 特になし |
| 再生布(緑色) | 特性発動時、累積ダメージ÷基準値の比率が最大のマスが回復する。回復量は12〜16の5通り等確率。ただし既に黄色枠に収まっているマスは除く。同評価のマスが2つ以上ある場合、対象はランダムで決定。**回復の適用タイミングは発動ターンの行動前**(=前ターンの行動後。例: 5ターン目の発動分は4ターン目の行動後〜5ターン目の行動前に適用) |
| 集中力変化・虹布(虹色) | 特性発動ごとに「消費集中力半減」と「消費集中力1.5倍で会心率上昇(固定値+24%)」が交互に来る。初回(5ターン目)は半減から始まる(5T=半減、9T=会心率アップ、13T=半減、…)。半減・1.5倍で端数が出た場合は**切り上げ** |
| 光布(ピンク布) | 特性発動時、黄色ゲージ内・赤ゲージ(残り-5以下)のマスを除いたランダムな1マスが発光(発動したターンのみ有効)。発光マスを縫うと会心率+24%(固定値加算)、マス補正2倍(しつけがけと重複時は4倍) |

**特性の発動タイミング:**

- 通常布を除く3種共通で、5ターン目に初めて発動し、以降4ターンごと(5, 9, 13, 17, …)に発動する。
- 「行動」と「ターン」は同義。精神統一・ぬいパワーシフト・しつけがけ・必殺技・糸ほぐしも行動(ターン)としてカウントする。
- 布タイプはこの4種(通常/再生/虹/光)で全て。

### 3.7 できのよさ判定

**マスごとの誤差算出:**

- 誤差 = |残り数値| を各マスで算出し、全マスの和を取る。
- 例外(黄色ゲージ外ペナルティ): あるマスの誤差が黄色ゲージ(基準値±4)の外にある場合、そのマスの誤差が9未満でも誤差9として扱う。誤差9以上の場合は実数値をそのまま使う。
  - 整理: マス誤差の評価値 = `誤差≤4 → 誤差` / `5≤誤差<9 → 9` / `誤差≥9 → 誤差`

**評価境界(マス数依存)** — 誤差合計が各境界値以下なら該当評価。★なし境界値を超えると失敗:

| マス数(該当部位) | ★3 | ★2 | ★1 | ★なし | 失敗 |
|---|---|---|---|---|---|
| 9マス(体上) | ≤8 | ≤17 | ≤36 | ≤49 | >49 |
| 7マス(ぬいぐるみ) | ≤5 | ≤17 | ≤27 | ≤30 | >30 |
| 6マス(体下・腕・ラグ) | ≤4 | ≤11 | ≤24 | ≤39 | >39 |
| 4マス(頭・足) | ≤2 | ≤7 | ≤16 | ≤29 | >29 |

- 大成功 = ★3。

## 4. シミュレータ機能仕様

### 4.1 データ駆動設計(最重要方針)

ゲーム動作に関わる数値・確率・ローテーションはすべてコードから分離し、編集可能なデータファイルに置く。仕様修正のたびにコード変更なしで反映できるようにする。

```
data/
  game-params.json      … 会心式係数、必殺基礎値、布特性の周期・回復量、評価境界 など
  needles.json          … 針ごとの会心率・集中度・開幕効果
  skills.json           … 特技: 消費・対象パターン・威力倍率・習得Lv
  concentration.json    … レベル別集中力テーブル
  recipes.csv           … レシピ(ユーザーがCSVで記入。フォーマットは data/README.md 参照)
```

**レシピCSVフォーマット(recipes.csv、1行=1レシピ):**

| 列 | 内容 |
|---|---|
| id | 一意キー(半角英数とアンダースコア) |
| name | アイテム名 |
| category | 部位/種別(体上・体下・頭・腕・足・ぬいぐるみ・ラグ) |
| cloth_type | 布タイプ(通常・再生・虹・光) |
| rows, cols | グリッド寸法(最大3×3)。部位ごとに固定(§3.1) |
| cell_r1c1 〜 cell_r3c3 | 9列。各位置のマスの基準値。マスが存在しない位置は空欄(7マス・6マス・4マスはこれで表現) |
| power_order | ぬいパワーサイクル。`/`区切りで先頭がターン1(例: `普通/強い/最強/？/弱い`)。トークンは 弱い・普通・強い・最強・？(「会心×2」はサイクルに現れないためトークンなし)。末尾まで進んだら先頭に戻ってループする |
| notes | 任意メモ |

- 文字コードは UTF-8(BOM付き。Excelでの編集を考慮)。

### 4.3 UI 素案

- 1画面構成: 左に布グリッド(マスごとの残り数値、黄色ゲージ内はハイライト)、右に「ターン数 / 現在ぬいパワー / 次パワー予告 / 集中力残」、下に特技ボタン群+行動ログ。
- **ぬいパワー表示は「現在のぬいパワー」と「次に来るぬいパワー」を並べて表示する**(今後2回分)。「次に来るぬいパワー」の定義:
  - 通常時 = 次ターンに実行されるぬいパワー(ぬいパワーシフト適用後の値。サイクル上の「？」は「？」のまま予告)。
  - 精神統一による固定中 = 固定が解除された後に最初に実行されるぬいパワー(固定期間中の据え置き分は表示に畳み込み、次の「異なる」パワーを示す)。ぬいパワーシフトを固定中に使うと、この表示が変わる(§3.3)。
  - 実装式(表示状態): `次パワー = 強制次パワー(シフト) ?? サイクル[(現在index+1) mod 長さ]`。次パワーが実行されるターン番号 = `現在ターン + max(1, 精神統一の残り固定ターン数)`。
- **布特性の発動ターンに当たるぬいパワーには、サイクル表示(全サイクルを並べた予告)の該当エントリに下線を引く**。各サイクルエントリについて、その「次に実行されるターン」が発動ターンなら下線を付す:
  - 次に実行されるターン = `現在ターン + ((i − 現在index + 長さ) mod 長さ)`(現在エントリ=現在ターン)。精神統一固定中は、現在エントリ以外に `固定残ターン − 1` を加算する(サイクル停止分の繰り下げ)。
  - **精神統一の固定中にイベントターンが来る場合は、現在エントリ(=固定中のぬいパワーの位置)に下線を付す**: 残り固定期間 `[現在ターン, 現在ターン+残−1]` に発動ターンが含まれるとき、その発動ターンの色で下線(固定は最長3ターン・発動間隔4ターンのため対象は高々1つ)。
  - 下線はパワー表示の枠(バッジ)に接しない**独立した直線**として、バッジの下に離して描画する。
  - 各エントリはサイクル周期ごとに再実行されるため、下線は**その次の実行機会**が発動ターンかで判定する。「？」エントリも実行ターン自体は確定するため判定可能。精神統一で実行ターンがずれるため、状態更新のたびに再計算する(都度更新)。
  - 下線色: 虹布の半減ターン=水色 / 虹布の会心(消費1.5倍)ターン=黄色 / 再生ターン(そのターンの直前に再生)=緑 / 光布の発光マスが出るターン=ピンク(発光は候補=残り数値5以上のマスがある場合のみ実発生。将来ターンは予告として下線を付す。§3.6)。通常布は下線なし。
- 操作フロー: **特技を選択→マスをタップ(対象範囲をハイライト表示)→同じマスを再タップで実行**。対象不要の特技(精神統一・ぬいパワーシフト・みだれぬい・無我の境地・しあげる)はボタン押下で即実行。**特技未選択でマスをタップした場合は「ぬう」を自動選択して同様に動作する**。
- 複数マス特技のアンカー(選択マス)規約: ヨコぬい=選択マスとその右 / 滝のぼり=**選択マスとその上** / たすきぬい=選択マスとその右上 / 逆たすきぬい=選択マスとその左上 / 水平ぬい・大滝のぼり=選択マスを中心に3マス / 巻きこみぬい=選択マスが中心。
- 誤差評価値(合計と★3ライン)は**常時表示**。
- ダメージ/回復は対象マス上に吹き出しで一時表示する(**約0.75秒**でフェードアウト): 通常ダメージ=オレンジ文字、会心=オレンジ背景に白抜き文字で強調、回復(糸ほぐし・再生布)=緑。**数値は符号なし・文字なし**(「会心!」等の文言は付けず、配色のみで区別)。
- しつけがけ中のマスは**色付き枠のみ**で表す(「しつけ×2」等の文字は表示しない)。光布の発光マスも**ピンク背景のみ**で表す(「発光」等の文字は表示しない)。
- 再生布の回復吹き出しは、同じ画面更新で発生するそのターンのダメージ吹き出しと**分けて表示**する(ダメージ吹き出しの表示後に回復分を表示)。
- 必殺チャージ発生時は、既存のバッジ表示に加えて**目立つ一時演出**(グリッド上への大きなオーバーレイ表示等。一時的なら現在の表示枠に収まらなくてよい)を行う。**配色はオレンジ系**とする。開幕効果(光の針)によるチャージでも表示してよい。
- デバッグモード時は、行動ログの各縫い行に**会心判定に用いた会心率**を表示する。
- 布タイプはマスの外側(グリッド背景)の色で表す: 通常=ベージュ / 再生=グリーン / 虹=イエロー / 光=ピンク。
- 結果表示のマス別誤差内訳(誤差評価の表)は**デバッグモード時のみ表示**する。デバッグモードは**URLクエリ `?debug=1`(ON)/`?debug=0`(OFF)で切替可能**(再ビルド不要。静的配信でも有効)。クエリ省略時は ui-config.json の `debugMode` をデフォルトとして使う(デフォルト false)。
- 行動ログは「特技名+対象+ダメージ+会心/頭打ち+布特性イベント」を1行ずつ記録。結果画面は★数・誤差内訳(ゲージ外9換算の明示)・使用ターン数・残集中力・リプレイコピー。
- 初期UIはPC前提。**スマホ版UIは表示領域を考慮してW1で抜本的に別設計とする**(現時点で考慮しない)。
- 検証モード(F5/F6): ヘッダーのトグルで切替(選択は localStorage に保存)。ONのとき以下を表示・操作可能にする。OFF(練習モード)ではすべて隠し、実機に近い感覚でプレイする。
  - **乱数シード**: 現在セッションのシードを表示。新規開始時にシードを指定できる(空欄なら自動生成)。同一シード+同一手順で結果は完全に再現される。
  - **アンドゥ**: 直前の行動を1つ取り消す(乱数状態ごと巻き戻す)。ゲーム開始直後まで繰り返し可能(=任意時点からのやり直し)。しあげる後(結果表示後)も取り消せる。
  - **実ロール値・会心判定**: 行動ログの各縫い行に、基礎値の出目(縫い12〜18/糸ほぐし6〜9)と会心判定に用いた会心率を付記する。表示はトグル切替で過去のログ行も含めて即時に切り替わる。
  - **リプレイ読込**: リプレイテキストを貼り付けて手順を再現する(下記)。
- リプレイ形式(F6、ARCHITECTURE A6): 「シード+設定+レシピid+行動列」のコンパクトJSON `{v:1, seed, recipeId, config, actions, check?}`。`check` は終了済みゲームの最終結果(★・誤差評価値・使用ターン数)で、読込時に再実行結果と照合し、不一致なら警告を表示する(仕様バージョン差の検出)。
  - コピー: 結果画面の「リプレイをコピー」ボタンで取得する。検証モード中はプレイ途中でもコピー可(その時点までの行動列)。
  - 読込(検証モードのみ): レシピidが現在の recipes.csv に存在しない場合はエラー。読込成功時はレシピ・針をリプレイの設定に合わせ、全行動を再実行して途中履歴ごと復元する(読込後も任意時点へアンドゥ可能)。
- パワー予告の表示/非表示も切替可能(実機準拠=非表示)。
- 設定項目: 針の種類と★数のみ(§1.1)。

## 6. 仮定一覧

5. **必殺チャージ判定は与ダメージ>0のターン終了時のみ**。「与ダメージ合計」には実際に適用された値(会心の頭打ち後)を用いる。チャージ保持中は再判定なし。
6. **ランダム選択は候補内で等確率**: みだれぬいの4マス選択、光布の発光マス、再生布のタイブレーク、「？」の変化先。

8. **複数マス特技に発光マスが含まれる場合、+24%・マス補正2倍はそのマスへの縫い分にのみ適用**。
9. **基礎値の乱数は縫いごと(複数マス特技ではマスごと)に独立して抽選**。
