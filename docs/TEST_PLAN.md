# ドラクエ10 さいほうシミュレータ テスト計画書 (v0.4)

作成日: 2026-07-03 / 状態: **承認済み (2026-07-03)**
関連文書: [SPEC.md](SPEC.md) / [ARCHITECTURE.md](ARCHITECTURE.md) / [DATA_DESIGN.md](DATA_DESIGN.md)

## 1. 方針

1. **テストの正は SPEC.md**。仕様書に書かれた数値・規則はすべて自動テストに落とし、実装との一致を機械的に担保する(転記ミス・実装ミスの両方を検出)。
2. テストの主対象は `core/` と `data/`(純関数のためテスト容易)。UIのテストは最小限とし、M2以降に動作確認ベースで行う。
3. 乱数を使うテストはすべて**シード固定**とし、不安定なテスト(flaky)を作らない。
4. フレームワーク: **Vitest**。高速なユニットスイート(常時実行)と、重い統計スイート(明示実行)を分離する。

## 2. テストスイート構成

| スイート | 実行 | 内容 |
|---|---|---|
| unit | `npm test`(常時・数秒) | T1〜T9、T11 |
| stats | `npm run test:stats`(明示実行・数十秒) | T10(大量試行の分布検証) |
| 受け入れ | 手動 | T12(実機突き合わせチェックリスト) |

## 3. テストケース体系

### T1. ダメージ計算(SPEC §3.2)

- **テーブル全数一致**: 縫い(特技倍率6種×パワー4種×基礎値7通り=168ケース)と糸ほぐし(パワー4種×マス補正2種×基礎値4通り=32ケース)を、SPEC §3.2 の表を期待値フィクスチャとして全数突き合わせ。
- 丸め規則: 正値切り上げ(3.5→4)・負値も正方向(-3.5→-3)・括弧ごとの適用。
- マス補正: 縫いはテーブル値×2/×4、糸ほぐしは括弧内(弱い×補正2が単純2倍と異なることを確認)。
- 会心: 2倍適用、累積ダメージ=基準値での頭打ち、頭打ちが起きない(2倍<残り)ケース、会心で縫いすぎが発生しないこと。
- 糸ほぐし: 会心判定が呼ばれないこと、回復の初期状態(累積ダメージ0)頭打ち(残り3に普通糸ほぐし→3回復)。

### T2. 会心率(SPEC §3.4)

- 式の各項の効果: コツ+1%、パッシブ実効+0.1%、ねらい倍率7、虹布会心ターン+24%、光布発光マス+24%、必殺×2、シフト会心×2、必殺+シフト併用で×4。
- ランダム会心(？→会心×2)に補正がないこと。
- 残り数値0以下のマス(縫い終わり・縫いすぎ)は会心判定なし(乱数消費なし)。
- 針7種×★4段の基礎値が needles.json と仕様書の表に一致すること。

### T3. 特技(SPEC §3.3)

- 全特技の消費集中力・特技倍率・対象オフセットが skills.json と仕様書の表に一致。
- 複数マス特技: マスごとに独立ロール・独立会心判定。
- 巻きこみぬい: 布端・欠けマスへのはみ出し無視(角・辺・ぬいぐるみの欠け位置)。
- みだれぬい: 異なる4マス選択(重複なし)、4マス布では全マス、倍率2/1/1/0.5の生成順、会心2倍適用後の値で降順ソート、ソート後の適用先が選択4マスであること。
- しつけがけ: 付与→1回縫われたら解除、重ね掛け不可、糸ほぐしにも補正2が乗ること(SPEC §3.2)。
- 精神統一: 3ターン固定、「？」ターンに使用した場合は抽選後のパワーが固定されること。
- ぬいパワーシフト: 指定パワーへの変更、「会心×2」選択時にシフト会心補正が付くこと。

### T4. ターン進行・ぬいパワー(SPEC §3.2)

- ぬいパワーサイクルのループ(末尾→先頭)。
- 「？」: ターン開始時(特技選択前)の抽選、他5種への変化。
- パワー係数(0.5/1/1/1.5/2)の適用。

### T5. 布の特性(SPEC §3.6)

- 発動タイミング: 5, 9, 13, … ターン。精神統一・シフト・しつけがけ・必殺技・糸ほぐしもターンとしてカウント。通常布は発動なし。
- 再生布: 対象=累積ダメージ÷基準値が最大のマス、黄色枠内は除外、同率はランダム、回復量12〜16。
- 虹布: 初回=半減、以降交互、半減/1.5倍の端数切り上げ(消費7→半減4、1.5倍11)、会心ターンの+24%。
- 光布: 候補=残り数値5以上のマスのみ(黄色ゲージ内・赤ゲージ除外)、発光は1マス・発動ターンのみ、+24%とマス補正2、しつけがけ重複で4倍、候補なしなら発光なし。

### T6. 集中力(SPEC §3.5)

- 開始値 = Lv80基礎値207 + 針の集中度(7種) + 奇跡開幕(30%判定で+30)。
- 不足時: コスト超の特技が使用不可、全特技不可でも「しあげる」(消費0)は可能。
- 自動回復: 残り10以下のターン開始時に10%で抽選。不発なら次の条件成立ターンも抽選されること、成功(+30)したら以後抽選されないこと(回復は1ゲーム1回)。
- 虹布の消費半減/1.5倍と特技コストの組み合わせ。

### T7. 必殺(SPEC §3.3)

- チャージ判定: 与ダメージ>0のターン終了時のみ、チャージ率式(基礎値×与ダメ×針会心率)、保持中は再判定なし。
- 光針の開幕チャージ(10%)。
- 無我の境地: チャージ状態でのみ使用可、消費0、1ターン消費(パワーが進む)、以降の会心率×2がゲーム終了まで持続、使用後の再チャージなし。

### T8. できのよさ判定(SPEC §3.7)

- マス誤差の評価値: `誤差≤4→そのまま / 5〜8→9 / ≥9→そのまま`(負の残り数値=縫いすぎ側も絶対値で同じ扱い)。
- 評価境界: マス数4種×境界4値について、境界ちょうど・境界+1の両側をテスト(例: 9マスで誤差合計8→★3、9→★2、49→★なし、50→失敗)。
- 誤差制限あり境界(SPEC v1.23): 誤差制限=1 のレシピについて 9/6/4マス×境界4値の両側をテスト(例: 9マスで誤差合計6→★3、7→★2)。7マス+誤差制限=1 は通常表にフォールバックすること。
- レシピ属性の検証(V9〜V11): craft_level 範囲外・equip_level 不正値・error_limit 0/1 以外が取り込みエラーになること。「-」は任意の部位で許容。

### T9. 決定論(ARCHITECTURE A3/A4)

- 同一シード+同一行動列 → 全ターンの状態・イベント・最終結果が完全一致(スナップショットテスト)。
- 乱数消費順(A4)のリグレッション: 消費順が変わると失敗するゴールデンテストを配置し、意図しない変更を検出する。
- リプレイJSON(A6)からの再実行が元のゲームと一致すること。

### T10. 統計的検証(statsスイート)

シード固定の大量試行(10万回オーダー)で分布を検証。判定は理論値±許容幅(3σ相当)。

- 基礎値: 縫い12〜18が各1/7、糸ほぐし6〜9が各1/4。
- 「？」の変化先が各1/5。
- 会心発生率が設定会心率に収束(ねらいぬい・補正込みのケース含む)。
- 再生布タイブレーク・光布発光対象・みだれぬい対象選択の等確率性。
- 奇跡開幕30%・光開幕10%・集中力自動回復10%。

### T11. データ検証(DATA_DESIGN)

- JSONファイル(game-params / needles / skills / concentration)の値が SPEC の数表と一致(フィクスチャ突き合わせ)。
- recipes.csv バリデーション V1〜V8: 正常9件が全件ロードできること+規則ごとの異常CSVでエラー/警告が出ること。
- 集中力テーブル: 80要素、Lv80=207。

### T12. 実機突き合わせ(手動・受け入れ観点)

ユーザーが実機と比較するためのチェックリスト(M2完了後に実施):

1. 任意レシピでダメージ出目が実機の観測範囲と一致するか(パワー別に数十サンプル)
2. 糸ほぐしの回復量(弱い=3,4 / 普通=6〜9)
3. 布特性の発動ターン(5,9,13…)と効果表示
4. 虹布の消費集中力表示(半減・1.5倍の端数)
5. できのよさ判定: 実機の完成品の誤差と★数の対応
6. 会心・必殺の発生頻度の体感差(厳密検証は将来の統計モードで)

## 4. フィクスチャ(期待値データ)の管理

- SPEC §3.2 のダメージテーブル等は `tests/fixtures/spec-tables.ts` に転記し、**「仕様書の表 → フィクスチャ → 実装」の三者一致**をテストで保証する。
- 仕様変更時は SPEC → フィクスチャの順に更新し、テストが実装の追従漏れを検出する運用とする。

## 5. マイルストーンとの対応

| マイルストーン | 完了に必要なテスト |
