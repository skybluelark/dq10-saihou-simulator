# レシピデータ (recipes.csv) フォーマット

1行 = 1レシピ。文字コードは UTF-8 (BOM付き、Excel編集対応)。

## 列定義

| 列 | 必須 | 内容 |
|---|---|---|
| id | ○ | 一意キー。半角英数とアンダースコア(例: `kenja_robe`) |
| name | ○ | アイテム名(例: `賢哲のころも`) |
| category | ○ | `体上` `体下` `頭` `腕` `足` `ぬいぐるみ` `ラグ` のいずれか |
| cloth_type | ○ | `通常` `再生` `虹` `光` のいずれか |
| rows | ○ | グリッド行数(1〜3)。部位ごとに固定: 頭・足=2、体上・ぬいぐるみ=3、体下=3、腕・ラグ=2 |
| cols | ○ | グリッド列数(1〜3)。部位ごとに固定: 頭・足=2、体上・ぬいぐるみ=3、体下=2、腕・ラグ=3 |
| cell_r1c1 〜 cell_r3c3 | ○ | 各位置のマスの基準値(正整数)。**マスが存在しない位置は空欄**。r=行, c=列(r1c1が左上) |
| power_order | ○ | ぬいパワーサイクル。`/` 区切り、先頭がターン1。トークン: `弱い` `普通` `強い` `最強` `？`(「会心×2」はサイクルに現れないためトークンなし)。末尾までいったら先頭へループ |
| notes | - | 任意メモ |

## 記入例(数値は架空のサンプル)

```csv
kenja_robe,賢哲のころも,体上,虹,3,3,52,58,52,60,64,60,55,62,55,普通/強い/最強/？/弱い,サンプル
nuigurumi_x,○○ぬいぐるみ,ぬいぐるみ,光,3,3,40,,40,45,50,45,42,42,,7マスの例(r1c2とr3c3が空)
hat_y,○○ぼうし,頭,通常,2,2,30,32,30,32,,,,,,弱い/普通/強い/最強,4マスの例(cell列は先頭4つのみ使用)
```

- 4マス(2×2)の場合: rows=2, cols=2 とし、cell_r1c1, r1c2, r2c1, r2c2 に記入、残りは空欄。
- 7マス・6マスなど欠けのある配置は、該当位置の cell を空欄にして表現する。
- マス数と評価境界(★3〜失敗)の対応はシミュレータ側で自動判定(9/7/6/4マス)。
